# 清单切换功能实现说明

## 功能概述

成功实现了三层级的系统清单切换功能，用户可以在不同的系统清单之间无缝切换，包括：

1. **平台清单** - 原有的IT平台系统
2. **系统清单** - 企业核心业务系统
3. **历史系统清单** - 历史遗留系统集合

## 实现的功能特性

### 🔄 清单切换
- **标签式切换**: 在页面头部提供清晰的标签按钮
- **动态加载**: 切换时动态加载对应清单的系统数据
- **状态保持**: 每个清单独立维护搜索和筛选状态
- **视觉反馈**: 当前激活的清单标签有明显的视觉标识

#### 历史系统入口
- 在"系统清单"中点击"历史系统"会自动跳转到"历史系统清单"
- 历史系统卡片有特殊的箭头徽章标识
- 支持动画效果提示用户这是一个入口

#### 系统状态显示
- **正常运行**: 绿色圆点
- **维护中**: 橙色圆点  
- **部分运行**: 深橙色圆点
- **已停用/已迁移**: 红色圆点

## 技术实现细节

### 数据结构
```javascript
// 清单类型枚举
SYSTEM_LIST_TYPES = {
  PLATFORM: 'platform',
  SYSTEM: 'system', 
  HISTORY: 'history'
}

// 清单配置
LIST_CONFIG = {
  [type]: {
    title: '清单标题',
    subtitle: '清单描述',
    data: '数据源'
  }
}
```

### 核心方法
- `switchList(listType)`: 切换清单类型
- `openSystem(system)`: 处理系统点击，包括历史系统特殊逻辑
- 动态渲染标签和内容

### 样式特性
- **响应式设计**: 移动端自适应
- **动画效果**: 平滑的切换动画
- **视觉层次**: 清晰的信息层级
- **交互反馈**: 悬停和点击效果

## 用户体验

### 🎨 界面设计
- 清单标签采用现代化的胶囊按钮设计
- 当前清单信息区域突出显示
- 历史系统入口有特殊的视觉标识
- 系统状态用不同颜色区分

### 🔍 搜索功能
- 每个清单独立的搜索功能
- 支持按系统名称、描述、分类搜索
- 切换清单时自动清空搜索条件

### 📱 移动端适配
- 标签按钮在小屏幕上自动换行
- 保持所有功能在移动端的可用性
- 优化触摸交互体验

## 使用说明

### 基本操作
1. **切换清单**: 点击页面顶部的标签按钮
2. **查看历史系统**: 在系统清单中点击"历史系统"卡片
3. **搜索系统**: 使用搜索框快速定位系统
4. **收藏系统**: 点击系统卡片右上角的星标

### 导航路径
```
平台清单 ←→ 系统清单 → 历史系统清单
```

### 系统访问
- 大部分系统点击后显示"即将打开"提示
- 集团官网直接跳转到 https://www.ynsnjt.cn
- 历史系统点击后跳转到历史系统清单
